# 4_Voice_Motion_Test 程序使用完整流程

本文档说明 `voice_motion_test` 从环境准备、交叉编译、上传部署、配置修改、运行测试到 PC/ROS/Web/AI 联动的完整流程。

## 1. 这个程序是什么

`4_Voice_Motion_Test` 是龙芯 2K0301 板端用户态程序，不是 ROS 节点，也不是系统镜像。

它的作用：

- 控制左右后轮电机。
- 控制前轮舵机。
- 读取左右编码器。
- 监听板载语音模块串口。
- 或作为 UDP agent，接收 PC/ROS 发来的运动命令。

部署方式是：

```text
PC 交叉编译 LoongArch 可执行文件
-> scp 上传到 2K0301 板子 /home/root/
-> 板子运行 /home/root/voice_motion_test
```

这不是“烧录固件”。只有重新烧系统镜像才叫烧录。

## 2. 电脑端需要的软件和工具

Ubuntu 22.04 上需要：

```bash
sudo apt update
sudo apt install -y build-essential cmake make openssh-client picocom v4l-utils
```

说明：

- `build-essential`：提供基础编译工具。
- `cmake`、`make`：构建程序。
- `openssh-client`：用 `ssh/scp` 连接板子。
- `picocom`：串口终端。
- `v4l-utils`：摄像头检测工具，可选但推荐。

ROS 2 实机 Web/AI 部分还需要你已经安装好的 ROS 2 Humble。

执行任何 ROS、colcon、Python ROS、Gazebo、交叉编译命令前，先退出 conda：

```bash
source ~/miniconda3/etc/profile.d/conda.sh 2>/dev/null || source ~/anaconda3/etc/profile.d/conda.sh 2>/dev/null || true
conda deactivate 2>/dev/null || true
```

## 4. 板子端默认文件位置

板子上主要文件：

```text
/home/root/voice_motion_test          # 板端可执行程序
/home/root/voice_motion_config.ini    # 运行配置
/home/root/voice_motion_mode          # run 或 agent
/home/root/voice_motion.log           # 运行日志
/etc/init.d/S99_voice_motion_test     # 开机自启脚本
```

当前常用模式：

```text
agent
```

`agent` 模式表示：小车等待 PC/ROS/Web/AI 通过 WiFi UDP 发控制指令。

`run` 模式表示：小车自己监听板载语音模块串口，不依赖 PC AI，但只能识别固定语音命令帧。

## 5. 什么情况需要交叉编译

只改配置文件，不需要交叉编译：

```text
4_Voice_Motion_Test/config/voice_motion_config.ini
```

例如修改这些参数，只需要上传配置并重启：

```ini
servo_center_deg=92
pulse_on_ms=40
pulse_off_ms=120
forward_counts=25
turn_counts=16
```

修改 C++ 源码才需要交叉编译：

```text
4_Voice_Motion_Test/src/voice_motion_test.cpp
```

例如改协议、改控制逻辑、改安全策略，就需要重新编译并上传新的 `voice_motion_test`。

## 12. 单独测试电机通道

注意：测试前把车轮架空。

停止 agent：

```bash
ssh root@192.168.43.192 '/etc/init.d/S99_voice_motion_test stop'
```

测 motor1：

```bash
ssh root@192.168.43.192 '/home/root/encoder_slow_roll_test swap1 8 1 0 2200 40 120'
```

测 motor2：

```bash
ssh root@192.168.43.192 '/home/root/encoder_slow_roll_test swap2 8 1 0 2200 40 120'
```

测两轮：

```bash
ssh root@192.168.43.192 '/home/root/encoder_slow_roll_test swapboth 8 1 0 2200 40 120'
```

恢复 agent：

```bash
ssh root@192.168.43.192 '/etc/init.d/S99_voice_motion_test start'
```

判断方法：

- 如果某一路编码器一直是 0，说明该路电机、驱动通道、线缆或机械传动可能有问题。
- 如果左右信号线和粗电机线都对调后，仍然固定某一侧轮不转，优先怀疑该侧电机或机械传动。

## 16. 常见问题

### 只改配置后为什么没生效

可能只改了电脑文件，没有上传到板子。需要执行：

```bash
scp 4_Voice_Motion_Test/config/voice_motion_config.ini root@192.168.43.192:/home/root/voice_motion_config.ini
ssh root@192.168.43.192 '/etc/init.d/S99_voice_motion_test restart'
```

### 改了 C++ 但板子行为没变

需要重新交叉编译并上传新的可执行文件：

```bash
cd ~/Library_Patrol_Project/4_Voice_Motion_Test
./build.sh
scp build/voice_motion_test root@192.168.43.192:/home/root/voice_motion_test
ssh root@192.168.43.192 'chmod +x /home/root/voice_motion_test && /etc/init.d/S99_voice_motion_test restart'
```

### 板子 ping 不通

检查电脑和板子是否在同一个热点或局域网。板子串口查看：

```bash
picocom -b 115200 /dev/ttyACM0
ifconfig wlan0
```

退出 picocom：

```text
Ctrl+A，然后 Ctrl+X
```

### Web 打不开

先看状态：

```bash
cd ~/Library_Patrol_Project
./run_real_robot.sh status
```

重启 Web：

```bash
./run_real_robot.sh stop
./run_real_robot.sh start-web
```

### 小车不动但 UDP 有回应

说明网络和板端程序是通的，优先检查：

- 电池电压。
- 电机驱动板电源输入。
- 电机粗线端子。
- 右/左电机是否卡住。
- 单通道测试 `swap1/swap2` 的编码器输出。
